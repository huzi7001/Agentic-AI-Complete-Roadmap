## 🧠 Learning Philosophy

I am following a **Pareto Principle (80/20)** approach:

- learn the **most important concepts first**
- build **real projects early**
- focus on **skills that are actually used in jobs**
- revise and improve continuously

Instead of only watching theory, I want to:

- understand concepts deeply
- practice with code
- build deployable projects
- maintain a strong GitHub profile
